# Development Documentation

## Project Structure

This folder contains project documentation and technical specifications.

### Files

- `game_design.md` - High-level game design and mechanics
- `technical_architecture.md` - System architecture and code organization
- `simulation_rules.md` - Rules and formulas for simulation
- `mvp_scope.md` - MVP scope and features
- `codex_tasks.md` - AI assistant prompts and tasks

## Getting Started

1. Read `game_design.md` for overall vision
2. Review `technical_architecture.md` for code structure
3. Check `mvp_scope.md` for current development phase
4. Reference `simulation_rules.md` for game mechanics

## Core Principles

- **Simulation-First**: All game logic in C#, separate from Godot rendering
- **Data-Driven**: Configuration in JSON files, not hardcoded
- **Modular**: Independent, testable systems
- **Scalable**: Designed for 1000+ simulated citizens
