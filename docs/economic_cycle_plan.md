# План замыкания экономического цикла

## 1. Цель

Сделать экономику ближе к реальному циклу, где деньги не накапливаются бесконечно на счетах, а регулярно проходят через жителей, бизнес и государство.

Базовый круг:

`бизнес производит -> жители покупают -> бизнес получает деньги -> бизнес платит зарплаты и налоги -> жители получают доход -> жители снова удовлетворяют потребности`

Государство участвует как перераспределяющий и инфраструктурный слой:

`налоги -> бюджет -> проекты/содержание/услуги -> качество жизни и условия для бизнеса`

## 2. Текущее состояние

- [x] Есть жители с доходом, работой и потребностями.
- [x] Есть бизнесы с сотрудниками, выручкой, расходами, выпуском и продажами.
- [x] Есть зарплаты, подоходный налог, налог на бизнес и госрасходы.
- [x] Есть денежный dashboard с притоками, оттоками, налогами и зарплатами.
- [x] Продажи жителям списывают деньги у жителей и начисляют выручку бизнесу; продажи внешнему рынку оставлены отдельным ограниченным притоком.
- [x] Потребности жителей требуют регулярных покупок по базовым категориям потребления.
- [x] У бизнеса есть отдельный денежный счёт, tick-бухгалтерия, уровень, качество продукта и базовый инвестиционный цикл.
- [x] Есть явный расчёт спроса со стороны жителей по базовым категориям потребностей.
- [x] Правила открытия/закрытия предприятий связаны с неудовлетворённым спросом, доступной нишей, рабочей силой и жизнеспособностью бизнеса.

## 3. Целевая модель денежных потоков

### 3.1. Источники денег

- [x] Стартовые деньги:
  - бюджет города;
  - стартовые денежные остатки бизнесов;
  - стартовые накопления жителей или домохозяйств.
- [x] Внешний рынок:
  - ограниченный внешний спрос на экспорт/излишки;
  - инвестиции извне;
  - миграционный приток с личными накоплениями;
  - государственные гранты или кризисная помощь.
- [x] Внутренняя экономика:
  - не создаёт деньги, а перераспределяет их между жителями, бизнесами и бюджетом.

### 3.2. Поступление денег в экономику

- [x] Ввести явное разделение:
  - `ExternalInflow` - деньги пришли извне;
  - `InternalTransfer` - деньги перешли между участниками города;
  - `ExternalOutflow` - деньги ушли из города.
- [x] Продажи жителям должны быть внутренним переводом:
  - деньги списываются с дохода/счёта жителя или домохозяйства;
  - бизнес получает эти деньги на счёт.
- [x] Продажи внешнему рынку должны оставаться внешним притоком:
  - ограничиваются типом бизнеса, производственным излишком и внешним спросом.

### 3.3. Распределение денег

- [ ] Бизнес распределяет деньги на:
  - зарплаты;
  - налоги;
  - закупки/операционные расходы;
  - инвестиции в развитие;
  - резерв ликвидности.
- [ ] Жители распределяют деньги на:
  - еду;
  - жильё;
  - услуги/здоровье;
  - досуг;
  - накопления.
- [ ] Государство распределяет деньги на:
  - проекты;
  - содержание инфраструктуры;
  - субсидии или кризисные меры;
  - внешние расходы.

### 3.4. Отток денег

- [x] Оттоком считать только деньги, покидающие локальную экономику:
  - импорт товаров/ресурсов;
  - внешние подрядчики;
  - содержание, не имеющее локального получателя;
  - кризисные потери;
  - долг/проценты, когда появится долговая модель.
- [x] Зарплаты, налоги, покупки у местных бизнесов и аренда не являются оттоком.

## 4. Структура данных

### 4.1. Деньги жителей

- [x] Переименовать/уточнить смысл `Citizen.Income`:
  - либо оставить как накопленный доход;
  - либо ввести отдельное поле `Cash`.
- [x] Для домохозяйств добавить агрегаты:
  - доступные деньги;
  - обязательные расходы;
  - discretionary spending;
  - сбережения.
- [x] Добавить лимиты расходов по потребностям, чтобы житель не покупал больше доступного бюджета.

### 4.2. Деньги бизнеса

- [x] Разделить бухгалтерию бизнеса:
  - `Cash` - реальные деньги на счёте;
  - `RevenueThisTick`;
  - `ExpensesThisTick`;
  - `ProfitThisTick`;
  - `TotalRevenue`;
  - `TotalExpenses`.
- [x] Считать банкротство по `Cash`, долговой нагрузке и серии убыточных тиков, а не только по накопленному `Revenue - Expenses`.
- [x] Добавить `BusinessLevel`.
- [x] Добавить `ProductQuality`.
- [x] Добавить `InvestmentReserve` или правило автоматического reinvestment.

### 4.3. Деньги государства

- [x] Сохранить `Budget` как кассу города.
- [x] Разделить расходы бюджета:
  - локальные закупки/зарплаты, если получатель внутри города;
  - внешний отток;
  - капитальные проекты.
- [x] В dashboard показывать, какие госрасходы остаются в городе, а какие уходят наружу.

## 5. Потребление жителей

### 5.1. Категории потребностей

- [ ] Связать потребности с категориями товаров/услуг:
  - еда -> farm/shop;
  - жильё -> rent/housing;
  - медицина -> clinic/service;
  - досуг -> park/service;
  - безопасность -> police/government service, чаще через бюджет, а не прямую покупку.
- [ ] Для каждой категории задать:
  - минимальный расход на тик/день;
  - желаемый расход при хорошем доходе;
  - эффект на удовлетворённость;
  - штраф при невозможности купить.

### 5.2. Алгоритм покупки

- [x] На каждом экономическом тике житель или домохозяйство формирует корзину потребностей.
- [ ] Система ищет доступные бизнесы по району и дорожной доступности.
- [x] Если товар/услуга есть и у жителя есть деньги:
  - деньги списываются у жителя/домохозяйства;
  - бизнес получает выручку;
  - потребность повышается или деградирует медленнее.
- [ ] Если денег нет:
  - покупка не происходит;
  - потребность падает;
  - растёт недовольство/кризисный риск.
- [x] Если доступного бизнеса нет:
  - покупка может уйти во внешний импорт с повышенной ценой;
  - либо потребность падает, если импорт отключён.

## 6. Производство бизнеса

### 6.1. Лимит производства

- [x] Ввести явный лимит выпуска за тик:
  - `ProductionCapacity = BaseOutput * BusinessLevelMultiplier * StaffingRatio * AccessMultiplier`.
- [x] `StaffingRatio` должен зависеть от количества нанятых сотрудников.
- [x] Если сотрудников мало, производство должно падать.
- [x] Если нет дорожного доступа, производство и продажи должны резко падать или обнуляться.

### 6.2. Спрос

- [x] Внутренний спрос рассчитывается от жителей:
  - количество жителей;
  - неудовлетворённость потребности;
  - доступность товара;
  - цена;
  - качество продукта.
- [x] Внешний спрос ограничен и не должен бесконечно покупать всё производство.
- [ ] `DemandMultiplier` заменить или расширить на:
  - `LocalDemand`;
  - `ExternalDemand`;
  - `QualityDemandBonus`;
  - `PriceDemandPenalty`.

### 6.3. Продажи

- [x] Продажи жителям ограничены:
  - производством;
  - спросом;
  - деньгами жителей;
  - доступностью бизнеса.
- [x] Продажи внешнему рынку ограничены:
  - остатком производства после локальных продаж;
  - экспортной способностью типа бизнеса;
  - внешним спросом.

## 7. Развитие предприятий

### 7.1. Уровень предприятия

- [x] Добавить уровни бизнеса:
  - 1 - базовый;
  - 2 - улучшенный;
  - 3 - развитый;
  - далее по необходимости.
- [ ] Уровень влияет на:
  - качество продукта;
  - производительность;
  - максимальный штат;
  - расходы на содержание;
  - привлекательность для работников.

### 7.2. Инвестиции

- [x] Бизнес может инвестировать часть прибыли в развитие.
- [x] Инвестиция возможна только если:
  - есть положительный cashflow;
  - сохранён минимальный резерв на зарплаты;
  - есть спрос или стратегическая причина расширяться.
- [x] После накопления инвестиционного порога бизнес повышает уровень.
- [ ] Игрок может косвенно влиять через проекты, налоги и инфраструктуру.

## 8. Открытие и закрытие предприятий

### 8.1. Открытие

- [x] Новое предприятие может открыться, если:
  - есть неудовлетворённый спрос;
  - есть доступный участок;
  - можно построить дорожный маршрут;
  - есть доступная рабочая сила;
  - экономика района не перенасыщена этим типом бизнеса.
- [ ] Источник стартового капитала:
  - [x] внешний инвестор;
  - накопления существующего бизнеса;
  - городская субсидия;
  - предприниматель-житель в будущей модели собственности.
- [x] Открытие должно создавать событие/маркер для игрока, если оно существенно меняет район.

### 8.2. Закрытие

- [x] Предприятие закрывается или банкротится, если:
  - несколько тиков не может платить зарплаты;
  - cash ниже минимального резерва;
  - нет спроса;
  - нет дорожной доступности;
  - нет сотрудников;
  - слишком высокие налоги/расходы.
- [x] При закрытии:
  - сотрудники становятся безработными;
  - здание может стать пустующим;
  - район получает экономический сигнал;
  - игрок видит событие.

## 9. Баланс против бесконечных накоплений

- [x] У жителей должны быть регулярные расходы, зависящие от дохода и потребностей.
- [ ] У бизнеса должны быть:
  - зарплаты;
  - содержание;
  - налоги;
  - инвестиции;
  - закупки/импорт при необходимости.
- [ ] У государства должны быть:
  - содержание инфраструктуры;
  - стоимость проектов;
  - кризисные расходы.
- [ ] Добавить мягкие лимиты накоплений:
  - сверхрезерв бизнеса направляется в инвестиции, дивиденды или расширение;
  - сверхдоход жителей повышает потребление/досуг/качество жилья;
  - бюджет без расходов может повышать ожидания жителей или политическое давление.

## 12. Критерии готовности

- [x] Деньги жителей уменьшаются при удовлетворении потребностей.
- [x] Деньги бизнеса растут от реальных продаж и уменьшаются от зарплат/расходов/налогов/инвестиций.
- [x] Деньги бюджета растут от налогов и уменьшаются от проектов/содержания.
- [x] Внешний приток ограничен и объясним.
- [x] Внешний отток отделён от внутренних переводов.
- [x] Предприятия производят не больше своего лимита.
- [x] Предприятия могут развиваться при прибыли и закрываться при устойчивой проблеме.
- [ ] Dashboard показывает, почему экономика растёт, стагнирует или сжимается.
